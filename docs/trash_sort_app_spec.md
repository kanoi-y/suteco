# Suteco 仕様書

## 1. 概要

**アプリ名:** Suteco

スマートフォンのカメラまたはテキスト入力を用いてごみの品目を特定し、選択された自治体における分別方法を表示するアプリケーションを提供する。

本アプリケーション「Suteco」は OSS として公開し、利用者がローカル環境で起動して使用することを前提とする。

---

## 2. 目的

- ごみの分別方法を手軽に確認できるようにする
- 自治体ごとに異なる分別ルールへ対応できるようにする
- 自治体データを差し替えることで、他地域にも拡張可能な構成にする

---

## 3. 想定利用者

- 自治体ごとの分別方法を確認したい利用者
- 対応自治体データを追加・更新したい OSS コントリビューター

---

## 4. 対応プラットフォーム

- iOS
- Android

開発基盤は Expo / React Native を前提とする。

---

## 5. 提供機能

### 5.1 自治体選択

- 利用者は初回起動時に自治体を選択できる
- 選択した自治体は端末内に保存する
- 設定画面から自治体を変更できる

### 5.2 ごみ品目の入力

利用者は以下のいずれかの方法で品目を入力できる。

- スマートフォンのカメラで撮影する
- 端末内の画像を選択する
- テキストで品目名を検索する

### 5.3 品目候補の提示

画像入力時は、認識結果に基づいて候補となる品目を複数件表示する。

- 候補は 1 件以上表示する
- 利用者は候補の中から正しい品目を選択できる
- 適切な候補がない場合はテキスト検索へ移動できる

### 5.4 分別方法の表示

選択された自治体および品目に対応する分別情報を表示する。

表示項目は以下とする。

- 品目名
- 分別区分
- 出し方
- 注意事項
- 公式情報への参照先（存在する場合）

### 5.5 テキスト検索

- 品目名で検索できる
- 別名、表記ゆれ、一般的な呼称でも検索できる
- 検索結果から品目を選択し、分別方法を表示できる

### 5.6 設定

- 自治体の変更
- データバージョンの確認
- データ更新の実行（対応時）

---

## 6. 非機能要件

### 6.1 オフライン利用

- 端末内に保存済みの自治体データに対する検索・表示はオフラインで動作する
- 画像認識機能は、採用する認識方式に応じてオンラインまたはオフラインで動作する

### 6.2 応答性

- テキスト検索結果は入力後すみやかに表示されること
- 分別情報は選択後すみやかに表示されること

### 6.3 拡張性

- 自治体データを追加してもアプリ本体の実装変更が最小限で済む構成とする
- 品目追加、別名追加、分別ルール変更に対応できること

### 6.4 保守性

- 自治体データはアプリ本体とは分離して管理する
- データ形式はバリデーション可能な構造とする

---

## 7. システム構成

### 7.1 クライアントアプリ

- Expo
- React Native
- TypeScript

### 7.2 ローカルデータ

- 端末内データベースとして SQLite を利用する
- 自治体データは JSON として取り込む

### 7.3 認識処理

画像認識処理は抽象化し、差し替え可能な構成とする。

想定する方式は以下。

- 外部 AI API を利用する方式
- 端末内モデルを利用する方式
- ダミー実装による開発用方式

アプリ本体は、認識結果として「品目候補一覧」を受け取る前提で設計する。

---

## 8. データ設計

### 8.1 基本方針

画像認識結果をそのまま分別ルールに結び付けるのではなく、共通の品目 ID に正規化してから自治体ごとの分別情報へマッピングする。

処理の流れは以下とする。

1. 入力画像または検索語から品目候補を得る
2. 共通品目 ID を特定する
3. 選択中の自治体における分別ルールを取得する
4. 利用者へ表示する

### 8.2 主なデータ

#### municipalities

自治体情報を保持する。

- id
- 表示名
- バージョン

#### items

共通品目情報を保持する。

- id
- 品目名
- 別名一覧
- 検索用キーワード

#### disposal\_rules

自治体ごとの分別情報を保持する。

- 自治体 ID
- 品目 ID
- 分別区分
- 出し方
- 注意事項
- 参照先 URL

## 10. 検索仕様

### 10.1 品目検索

- 品目名の完全一致に対応する
- 別名、表記ゆれ、略称に対応する
- 部分一致検索に対応する

### 10.2 候補決定

- 画像認識時は複数候補を表示する
- 利用者の選択結果を最終決定とする

---

## 11. 自治体データ仕様

### 11.1 データ管理方針

- 自治体データは独立したファイルとして管理する
- データ形式は共通スキーマに従う
- データ追加時はスキーマ検証を通過する必要がある

### 11.2 データに含める項目

各品目について、少なくとも以下を定義する。

- 品目 ID
- 品目名
- 検索用別名
- 分別区分
- 出し方
- 注意事項
- 参照先 URL（任意）

---

## 12. エラーハンドリング

- 認識結果が取得できない場合は、テキスト検索への導線を表示する
- 対応する分別データが存在しない場合は、その旨を表示する
- 自治体未選択時は、自治体選択を促す
- データ読み込みに失敗した場合は再試行導線を表示する
